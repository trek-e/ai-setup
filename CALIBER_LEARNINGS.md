# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.

- **[pattern]** For demo GIF recordings of `caliber score`, use `Set FontSize 14`, `Set Width 1200`, `Set Height 900` to ensure the full score table (all 6 categories + summary) fits without clipping. Smaller terminals (1000x850 at font 16) cut off the bottom.
- **[pattern]** If merging to `next` fails from the main checkout, use the project's `next` worktree to pull/merge/push.
- **[correction]** Reproduce `caliber init` issues in the user's actual target repo (`~/Development/infra-core`), not in the `caliber` source repo.
- **[gotcha]** With `agent --print --output-format stream-json`, stdout is an event stream (`system`, `thinking`, `assistant`, `result`), and the structured payload is in the `result` event's `result` field (JSON string). Do not parse the full mixed stream as one JSON document.
- **[gotcha]** Do NOT use `Set FontFamily` in VHS tape files â€” VHS runs a headless browser and custom fonts (e.g. "Menlo") may not be available, causing the recording to hang indefinitely or fall back to a wide monospace font that wraps text.
- **[pattern]** For demo GIF recordings of `caliber score`, use `Set FontSize 12`, `Set Width 900`, `Set Height 1100` â€” the score output is ~64 lines tall, so a tall narrow terminal fits everything without clipping. Previous settings (1200x900 at font 14) cut off the bottom.
- **[gotcha]** The macOS `timeout` command is not available by default, and `gtimeout` (from `coreutils`) may not be installed. For VHS recordings, omit the timeout wrapper â€” VHS will complete on its own based on the tape's Sleep directives.
- **[gotcha]** The GitHub repo name (`ai-setup`) differs from the local directory and npm package name (`caliber`). When using `gh` CLI commands, the remote repo is `caliber-ai-org/ai-setup`, not `caliber-ai-org/caliber`.
- **[gotcha]** The GitHub repo name (`caliber-ai-org/ai-setup`) differs from local dir/package name (`caliber`). Both `rely-ai-org/caliber` and `caliber-ai-org/ai-setup` resolve to the same repo via `gh` CLI â€” use either, but be consistent.
- **[pattern]** When triaging duplicate PRs that modify the same file, merge them in dependency order: pure refactors first (e.g., ESM migration), then bug fixes, then features â€” this avoids merge conflicts between overlapping changes.
- **[gotcha]** `gh pr checkout` fails if you have uncommitted changes â€” run `git stash` first, then `gh pr checkout <number>`, and `git stash pop` when returning to the original branch.
- **[pattern]** For PR review workflow: checkout each PR â†’ `pnpm install --frozen-lockfile` â†’ `pnpm run build` â†’ `pnpm test` â†’ `gh pr review --approve` â†’ `gh pr merge --merge`. Always verify build+tests locally before approving.
- **[gotcha]** `gh project create` requires the `project` OAuth scope which isn't in the default `gh` token. Running `gh auth refresh -s project,read:project` prompts for browser auth â€” this can hang in non-interactive terminals.
- **[pattern]** When closing duplicate/inferior PRs, always leave a comment thanking the contributor, naming which PR was chosen instead, and explaining the specific technical reason (e.g., "#42 uses console.warn which is testable; yours uses process.stderr.write which can't be spied on in vitest").
- **[gotcha]** Issues linked to PRs may auto-close when the PR merges (if PR body has "Fixes #N"). Check with `gh issue view` before manually closing â€” you'll get "is already closed" if it was auto-closed.
- **[pattern]** For demo GIF recordings of `caliber score`, use `Set FontSize 12`, `Set Width 900`, `Set Height 1100` â€” the score output is ~64 lines tall, so a tall narrow terminal fits everything without clipping.
- **[gotcha]** VHS recordings that invoke LLM-backed commands (`caliber init`, `caliber refresh`) can hang indefinitely in the headless browser terminal. Pre-stage results using git tags (e.g., `before-init`, `after-init`) and only run deterministic commands (`caliber score`) live in the tape. For LLM commands, use `expect` scripts outside VHS or pre-commit the results.
- **[pattern]** For demo recordings, use `expect` (macOS built-in at `/usr/bin/expect`) to automate `caliber init` interactive prompts â€” match on prompt text like `"coding agents"`, `"session learning"`, `"What would you like to do"` and send `"a"` (select all) or `"\r"` (accept default). Set timeout to 180s for LLM-heavy flows.
- **[gotcha]** Always run VHS from the caliber source directory, not the demo repo â€” VHS `Output` paths are relative to CWD, and running from the wrong directory puts output files in unexpected locations.
- **[gotcha]** On macOS, common shell aliases like `rm -i` and `cp -i` cause `rm` and `cp` to prompt interactively, which hangs non-interactive shells. Use backslash-escaped commands (`\rm -f`, `\cp`) to bypass aliases in scripts and automated recordings.
- **[fix]** Before re-recording VHS tapes that switch git branches in the demo repo, clean the repo state first (`git checkout -- . && git clean -fd -q`) â€” otherwise dirty working tree from a previous recording causes checkout failures that silently break the tape.
- **[gotcha]** `npx create-video@latest` (Remotion scaffold) prompts interactively for options like TailwindCSS, which hangs non-interactive shells. Instead of using the scaffold, manually create `assets/video/` with `package.json`, `tsconfig.json`, and component files.
- **[pattern]** The Remotion video project at `assets/video/` has its own `package.json` and `pnpm-lock.yaml` â€” run `pnpm install` inside `assets/video/` before building. Build commands: `remotion render CaliberDemo assets/demo.mp4 --codec h264` for MP4, `--codec gif --every-nth-frame 2` for GIF.
- **[pattern]** When closing duplicate/inferior PRs, always leave a comment thanking the contributor, naming which PR was chosen instead, and explaining the specific technical reason.
- **[fix]** `gh pr create` without `--repo` fails when git remote points to `rely-ai-org/caliber` but the actual GitHub repo is `caliber-ai-org/ai-setup` â€” always use `--repo caliber-ai-org/ai-setup` or update the remote URL first with `git remote set-url origin`.
- **[gotcha]** GitHub `<video>` tags in README.md do NOT render in the main repo view â€” GitHub strips `<video>` from rendered markdown. Use `<img src="...gif">` for animated content in README, or link to the MP4 separately. The `<video>` tag only works in GitHub Issues, PRs, and Discussions.
- **[gotcha]** `WebFetch` from `raw.githubusercontent.com` can silently return empty responses for some SVG files (e.g., `openai.svg`). Use `cdn.jsdelivr.net/npm/simple-icons@latest/icons/<name>.svg` via `curl` as a reliable fallback for fetching simple-icons SVGs.
- **[gotcha]** Next.js RSC pages return serialized component data via `WebFetch`, not rendered HTML â€” you cannot extract Tailwind classes or DOM structure. To get design tokens, fetch the CSS file directly from the `/_next/static/chunks/*.css` URL found in the page source.
- **[gotcha]** Running `pnpm test` or `npx vitest` produces empty/silent failures when `pnpm install` hasn't been run after a fresh checkout or branch switch. Always run `pnpm install` before running tests if dependencies might be stale.
- **[pattern]** Scoring checks use `git check-ignore` (not `git ls-files`) to detect gitignored directories â€” it's a targeted check on specific paths rather than listing all tracked files, which scales better on large repos.
